# Contributing

Thank you for contributing to **Awesome AI Scientists**.

## What belongs here

Resources should be directly relevant to building, evaluating, or governing “AI Scientist” systems, including:

- Surveys and foundational references
- Agent architectures and tool-use patterns
- Benchmarks, datasets, evaluation frameworks
- Reproducibility and research engineering
- Safety, governance, and misuse resistance

## Submission rules

- One resource per bullet.
- Use this format: `- [Name](link) - One-sentence description.`
- Keep descriptions factual and concise.
- Prefer primary sources (papers, official docs, reputable repos).
- Avoid duplicate entries; improve an existing entry instead.

## Quality bar

A good entry has:

- A stable link
- A clear name
- A description that explains why it matters
- A sensible section placement

## How to contribute

1. Fork the repository.
2. Create a branch: `feature/<short-name>` or `fix/<short-name>`.
3. Make changes and ensure `awesome-lint` passes.
4. Open a pull request with a short rationale.

## Optional: add categories

If you propose a new section, explain:

- The gap it fills
- Why it cannot fit an existing section
- How many high-quality entries you expect to add soon
